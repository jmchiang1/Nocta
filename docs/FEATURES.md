# Nocta — Feature spec (v1)

## Scope summary

| Surface | In v1 | Out of v1 |
|---|---|---|
| **Tonight tab** | Hero why-card, AHI primary metric, secondary metrics row, night timeline, Morning Check-In, pattern card | Multi-night compare, wearable overlays |
| **Trends tab** | 7/30/90d ranges, metric trend charts, journal correlation cards, best/worst night view | Year view, cohort comparison |
| **Therapy tab** | Device pairing status, equipment lifecycle, view-only settings, doctor PDF export, compliance projection | DME reorder integration, telehealth booking |
| **You tab** | Profile, journal history, settings, account | Social features, sharing |
| **AI Coach** | Persistent FAB, context-aware chat opening from any card, suggested follow-ups | Voice input (deferred), proactive notifications |
| **Onboarding** | 8 screens, deferred signup, deferred CPAP pairing, health-data priming | Wearable connect, doctor invite |

## Feature 1: Causal "why" cards

The single highest-leverage feature. Every Tonight visit opens to one of these cards at the
top of the screen.

### Card states

| State | Trigger | Visual treatment |
|---|---|---|
| **Anomaly** | Today off-baseline by ≥1.5× std-dev on AHI / leak / SpO₂ / hours | Full hero, peach accent left rail, mini-sparkline, one action chip |
| **Steady** | Today within personal baseline range, no flagged events | Compact card, sage accent, no action chip, "Last night looked like your usual baseline" |
| **Win** | Today measurably better than 14d avg, with attributable cause | Compact card, sage accent, attribution sentence if confidence ≥ medium |
| **Escalation** | CAI > 5/hr in OSA patient, SpO₂ < 88%, persistent AHI > 5 over 14 nights after hygiene fixes | Coral accent rail, headline names the pattern without diagnosing, primary chip = "Prepare doctor summary" |
| **Insufficient data** | First 1–3 nights, session < 2 hrs, missing SleepHQ data | Neutral card, no accent, "Not enough data yet to spot patterns" |

### Card anatomy

One verdict, one action. Forensic detail is collapsed into a tappable receipts row —
never a wall of text on first read.

```
┌──────────────────────────────────────────────────┐
│ ✦ NOCTA COACH · LAST NIGHT                        │  Eyebrow — who/when, state-coloured
│                                                   │
│ Your AHI nearly doubled last night.               │  Headline — one verdict, ≤9 words
│                                                   │  Fraunces 30pt, italic on one word
│ ▁▁▃▂▇▇▆▄▂▁▁▁                                      │  Sparkline — 32px, carries timing
│                                                   │
│ → Try sleeping on your side tonight.              │  Action box (omit: steady, low-data)
│                                                   │
│ ──────────────────────────────────────────────   │
│ 10.5 events · mostly central · 2–5 a.m.       ⌄   │  Receipts row — tap to expand to
│ ──────────────────────────────────────────────   │  the observation paragraph
│ ▮▮▯  Medium confidence         Not medical advice │  Trust footer
└──────────────────────────────────────────────────┘
```

No chip row — the Coach FAB carries the chat affordance globally.

### AI insight JSON schema (Structured Outputs, strict mode)

Structured Outputs (`response_format: { type: "json_schema", strict: true }`).

### Prompt architecture

Don't pass raw waveform data to the LLM. Compute aggregates server-side, pass a summary blob.

**System prompt outline:**
```
You are Nocta Coach, a wellness companion that explains CPAP therapy data
to sleep apnea patients. You are NOT a medical device and do not give
medical advice.

Rules you never break:
- Never recommend pressure changes. Pressure changes are physician-only.
- Never name a diagnosis or condition (UARS, CSR, complex apnea, TECSA, etc).
- Never tell a user to stop or modify their therapy.
- Hedge causal claims with "this pattern often suggests" or "commonly indicates".
- Use second person, present tense, ~7th grade reading level.
- Output exactly ONE recommended action. Never a list.
- Cite specific numbers from the data blob provided. Never fabricate values.

If you detect ANY of the following in the data, set escalation_flag = "hard":
- Central apneas trending up (CAI > 5/hr appearing in OSA patient)
- SpO2 below 88% for any sustained period
- Persistent AHI > 5 over 14+ nights after equipment fixes
- Sudden new pattern of arrhythmia or rapid HR shifts

Output ONLY valid JSON matching the schema. No prose outside JSON.
```

**User message structure:**
```
<context>
  <user_profile>
    diagnosis: OSA
    prescribed_pressure_range: 5-12 cmH2O
    mask: ResMed AirFit P30i (Pillow)
    machine: AirSense 11 AutoSet
    therapy_start: 2025-05-27
  </user_profile>

  <last_night_summary>
    date: 2025-06-07
    session_hours: 6.2
    ahi: 10.5
    cai: 7.2  // central apnea index
    oai: 3.3  // obstructive apnea index
    leak_p95: 32
    pressure_median: 5.6
    pressure_p95: 6.6
    pressure_max: 6.6
    snore_index: 5
    spo2_min: 91  // if connected
  </last_night_summary>

  <event_clusters>
    [
      {"type": "CSA", "count": 14, "window": "02:01-05:14", "position_during": "stomach"},
      {"type": "OSA", "count": 2, "window": "04:22-04:49", "position_during": "back"},
      {"type": "leak_spike", "count": 4, "window": "03:18-04:32", "max_leak": 38}
    ]
  </event_clusters>

  <baseline_14d>
    ahi: 5.0
    cai: 1.2
    leak_p95: 11
    pressure_p95: 7.1
  </baseline_14d>

  <journal_today>
    morning: ["tired", "dry mouth"]
    yesterday_evening: ["alcohol_1_drink", "late_meal"]
  </journal_today>

  <journal_pattern>
    [
      "stomach_sleep_nights": {"count": 5, "avg_ahi": 8.4, "vs_other": "+2.3x"},
      "alcohol_nights": {"count": 7, "avg_ahi": 6.1, "vs_other": "+38%"}
    ]
  </journal_pattern>

Generate the why-card insight JSON for last night.
```

### Post-generation validation

Run programmatically before display:

1. **Citation check.** Every numeric value in `observation` must appear in the data blob or
   be derivable from it (e.g., a delta). Reject + retry if a number doesn't match.
2. **Forbidden-phrase scan.** Reject if output contains: any cmH2O value other than what's
   in the user's prescribed range as a description (never a recommendation); the word "you
   have" followed by a condition; the words "stop," "lower," "increase your pressure";
   condition names from a blocklist.
3. **Escalation second-pass.** If `escalation_flag != "none"`, send the output to a smaller
   second model for a sanity check.

### Tone examples

**Bad headline:** *"Bad night — see your doctor"* (vague, alarmist)
**Bad headline:** *"Great job sleeping 6.2 hours!"* (encouraging but ignores the AHI)
**Bad headline:** *"You may have central sleep apnea"* (diagnosis)

**Good headline:** *"Your AHI rose to 10.5 — most events were central, clustered between 2 and 5 a.m."*
**Good headline:** *"Last night looked like your usual baseline."* (steady state)
**Good headline:** *"This is the 4th night this week with rising central events. Worth bringing up with your sleep doctor."* (escalation, no diagnosis)

## Feature 2: Sleep journal

A tag-based contextual log, not a free-text diary. Entry is via the existing Morning
Check-In, expanded.

### The check-in flow

Three quick screens, ~20 seconds total. Each is multi-select chips with one "skip" option.

**Screen 1 — How do you feel this morning?**
- Rested · Tired · Sore · Foggy · Anxious · Congested · Sick · Headache

**Screen 2 — Last night I had…**
- Alcohol · Caffeine after 2pm · Big or late meal · Cold/flu meds · Nothing unusual

**Screen 3 — Yesterday I…**
- Exercised · Worked late · Traveled · Felt stressed · Changed my mask · Nothing unusual

Optional fourth screen, free-text: *"Anything else worth noting?"* — surfaces only on user
request.

### Bedtime tagging (optional, opt-in)

A tiny single-screen prompt at the user's usual sleep time, three chips: *"Anything to log
before bed?"* — alcohol, meal, stress, none. Skippable forever.

### Journal storage model

```json
{
  "date": "2025-06-07",
  "entries": [
    {
      "type": "morning",
      "timestamp": "2025-06-07T06:42:00Z",
      "tags": ["tired", "dry_mouth"]
    },
    {
      "type": "bedtime",
      "timestamp": "2025-06-06T22:30:00Z",
      "tags": ["alcohol_1_drink", "late_meal"]
    }
  ],
  "free_text": null
}
```

Tags are a controlled vocabulary. Don't let the LLM hallucinate new tag types — the UI
provides the canonical list.

### Correlation engine

A simple statistical job, not an LLM. Runs nightly server-side. For each tag the user has
logged ≥5 times in the last 30 days, compute:

- Average AHI on tagged-nights vs non-tagged-nights
- Average leak P95 on tagged vs non-tagged
- Average hours used on tagged vs non-tagged

Emit a **pattern card** when:
- Sample size ≥ 5 nights on each side
- Effect size ≥ 25% relative difference
- The tag isn't logged on > 80% of nights

Pattern cards appear:
1. On Home, below the night timeline, sage accent.
2. On Trends, as a dedicated "Patterns" section.
3. Embedded in why-cards when the AI determines the journal context explains the anomaly.

### Visual treatment of journal entries

In the night timeline (Home), journal tags from the night before appear as small peach
circle icons above the timeline at their approximate time. Tap = expand.

In the Trends view, the journal is a separate filterable lens.

## Feature 3 (supporting): Nocta Coach chat

Not a tab. A persistent floating action button (FAB) on every screen, plus inline "Ask
Nocta about this" chips on every metric card and insight card.

### Context-aware opening

When opened via a chip, the Coach receives the card's context as a system message. When
opened cold from the FAB, the Coach offers three suggested prompts based on recent data:
- *"Why was last night different?"*
- *"How can I reduce my leak rate?"*
- *"What's my best night this week and why?"*

### Coach safety rails

Same hard rules as the insight schema. Additionally:

- The Coach answers factual CPAP-domain questions from a curated RAG knowledge base.
- If the user asks the Coach to recommend a pressure change, mask change, or diagnosis:
  respond with a kind redirect and offer to help prepare a doctor summary instead.
- Every Coach response ends with a tiny "Observation, not medical advice" footer.

## Feature 4 (supporting): Equipment lifecycle tracker

Lives on the Therapy tab. User enters during onboarding (or later): mask model + cushion
install date, filter, tubing, water chamber, headgear install dates.

Nocta shows a small visual aging strip per item with replacement recommendations:
- Cushion: 30 days
- Mask frame: 90 days
- Filter (disposable): 30 days
- Tubing: 6 months
- Water chamber: 6 months
- Headgear: 6 months

Aging items show an amber state at 80% lifespan, coral at 100%.

---

## Feature 5 (supporting): Doctor PDF export

One tap on the Therapy tab. Generates a clean, clinician-formatted summary of the last
30 / 90 days: AHI trend chart, event breakdown (CSA / OSA / hypopnea counts), leak P95
trend, pressure range used, compliance summary. Neutral footer: *"Generated by Nocta from
SleepHQ data. Not a medical record."* No AI insights in the export.

## Onboarding (8 screens)

| # | Screen | Purpose |
|---|---|---|
| 1 | Welcome | Brand moment, sets warm tone |
| 2 | Goals (multi-select chips) | "What brings you to Nocta?" |
| 3 | About you | Age range, sex (optional), prescribed pressure range (optional) |
| 4 | Health-data priming | Explains *why* Nocta wants HealthKit/HealthConnect access |
| 5 | CPAP pairing | SleepHQ OAuth, with "I'll do this later" fallback |
| 6 | Equipment setup | Mask model, install dates — feeds the lifecycle tracker |
| 7 | Tomorrow's expectations | Illustrated explainer of the first insight |
| 8 | Notification + account | Account creation deferred until value has been earned |

Drop a one-time medical disclaimer modal on first sign-in.

---

## Things to deliberately omit from v1

- **Wearable integration.** Defer to v1.1.
- **Best/worst night side-by-side compare.** Defer (a light version may land in Trends).
- **Custom home metrics.** With one primary + three secondary, customization adds confusion.
- **Streaks / gamification.** Active no.
- **Community / shareable links.** Defer.
